# UI Rules

## AI Judgment Display

Every AI judgment must show:

- evidence,
- uncertainty/confidence,
- recommended next action,
- reviewer action,
- audit/log status when relevant.

## States

- Empty states explain missing data or permission.
- Error states are actionable.
- Policy failures are explicit and not hidden behind generic errors.
- High-risk actions must show approval state and cannot be visually styled as ordinary buttons.
