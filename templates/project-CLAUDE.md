# Project Operating Rule: Harness Engineering

This project uses Harness Engineering.

Do not treat requests as one-off prompts when they affect product, architecture, code rules, AI Agent behavior, evals, or handoff. Update the harness artifacts instead.

## Required order

1. Preserve source.
2. Normalize meeting/input.
3. Frame CPS.
4. Define principles.
5. Define vocabulary.
6. Build domain model.
7. Write PRD and feature spec.
8. Design architecture and dataflow.
9. Design AI Agent loop.
10. Define code rules.
11. Define evals.
12. Update change log.
13. Prepare handoff.

## Final report format

- Files changed
- Decisions made
- Assumptions
- Validation performed
- Remaining risks
- Next action
