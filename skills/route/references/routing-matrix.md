# Routing Matrix

Use the earliest missing artifact to decide next action. If multiple are missing, prioritize in this order:

1. Intake/source preservation.
2. Business model when value, data, costs, or revenue are unclear.
3. CPS.
4. Principles/definitions.
5. Domain model.
6. PRD/feature spec.
7. Architecture/agent loop.
8. Code rules.
9. Evals.
10. Risk/compliance.
11. Demo/handoff.

Do not jump to implementation when the central domain object or approval state is undefined.
