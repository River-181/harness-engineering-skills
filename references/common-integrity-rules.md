# Common Integrity Rules

Apply these rules to every Harness Engineering skill:

- Inspect existing artifacts before claiming their status.
- Prefer updating existing artifacts over creating parallel duplicates.
- Do not overwrite user work without naming the target files and the intended change.
- Put unresolved ambiguity in `harness.yaml.open_questions` or the relevant artifact.
- For regulated, customer-facing, or financial workflows, require evidence, approval, audit logging, and fail-closed behavior before demo or execution.
- End with the common completion report in `references/common-completion-report.md`.
